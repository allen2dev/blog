团队里最常见的一句是「我觉得这次好一点」。**体感在迭代前几周有用，之后就变成了噪声**。要做可持续迭代，至少需要：

1. **固定问题集（golden questions）**：不必多，但要覆盖主干分支与已知翻车点。
2. **可重复的评分脚本**：同一输入跑两遍，输出差异可 diff——人力只需要判「是否变差」。
3. **最简单的可视化**：表格列出问题、模型版本、得分或标签，足以在评审会上对齐。

前端在这里的价值常被低估：你们最擅长**把抽象分数变成可排序的表格、把两次回答并排展示**。这比后端多写一个 REST 更能加速决策。

---

## 评测不是为了炫技

是为了在模型提供商偷偷更新权重、或者你们换了 prompt 的那一天，你能在十分钟内知道：**有没有大面积退步**。没有这一步，所谓「AI 功能」只是礼品包装的技术债。

我见过最务实的起点，是一个 markdown 文件列出二十个问题、一个 npm script 调接口把回答落盘——复杂仪表盘可以以后再说。**先有基线，再谈优化**。
