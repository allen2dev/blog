组件化浪潮起来之后，JavaScript 模块边界清晰了，**样式却仍然像煤气泄漏**——一处改 `h3`，三页排版一起歪。那段时间我们试过几种药方：

**BEM** 把命名约定强行掰长：`block__element--modifier`。冗长，但在没有构建魔法的世界里，它至少让 grep 变得有意义。**Less/Sass** 带来了嵌套与变量，却没自带「作用域」——滥用嵌套的人往往只是把特异性的问题推迟到下一个 sprint。

后来 **CSS Modules**、scoped style、原子类框架依次登场。争论哪种「更正统」意义不大，更值得问的是：**样式的所有权是谁？**

---

## 我至今仍遵守的两条规则

1. **同一组件目录内的样式默认不外溢**；若必须全局（字体、reset），集中在一个入口并文档化。
2. **魔法数字进变量**：颜色与间距若散落八十处，设计一改你就输了。

```css
/* 举例：用自定义属性集中约束一层 */
.article {
  --gap: 1.25rem;
  --muted: #666;
}
```

回头看，CSS 最难的不是选择器优先级，而是**团队能否共享同一套「什么算破坏性行为」**。这和 TypeScript 一样：工具只能放大共识，不能代替共识。
