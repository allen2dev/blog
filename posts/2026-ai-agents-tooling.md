近两三年最明显的变化，不是某个新框架，而是 **「模型 + 工具调用」** 成了默认心智：查文档、跑命令、改文件，都可以被调度成一步。

作为写过十年脚本的人，我看到的机会是：

- **把确定性留给脚本**：格式检查、构建、发布到静态托管，仍然应该是可重复的命令。
- **把探索性留给模型**：接口调研、错误日志归纳、生成初版 CRUD。
- **边界用契约锁住**：OpenAPI、类型定义、CI 红灯，Agent 再聪明也不能绕过。

```bash
# 我的工作台长期维持「一条命令见真章」
npm run build && npm run preview
```

未来会更吵（更多 agent、更多 MCP server），**能分清何时该自动化、何时该人肉决策的人**，会更吃香——这和 2016 年说「先想清楚再写代码」其实是同一句。
